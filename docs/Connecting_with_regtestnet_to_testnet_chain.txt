Step 4 Objectives

The goal of Step 4 is to confirm:

Bitcoin Core is running in TESTNET mode

The node is using the correct testnet data directory

The node is connected to real public peers

Live blockchain data is being exchanged

On-chain activity is observable and verifiable

Testnet Configuration Validation

Bitcoin Core was started explicitly in testnet mode:

bitcoind -testnet -daemon

Verification command:

bitcoin-cli -testnet getblockchaininfo

Expected result:

"chain": "test"

This confirms the node is running on Bitcoin Testnet and not regtest or mainnet.

Testnet Data Directory Verification

Bitcoin Core maintains separate directories for each network.

For testnet, the active directory is:

~/.bitcoin/testnet3/

The presence of the following confirms correct setup:

blocks/

chainstate/

debug.log

wallets/

This ensures:

No overlap with regtest data

Independent blockchain state

Clean separation of environments

Live Network Connectivity (Key Validation)

The most important confirmation is real peer connectivity.

Live logs were monitored using:

tail -f ~/.bitcoin/testnet3/debug.log

Observed indicators of real network participation:

Outbound and inbound peer connections

Header synchronization messages

UpdateTip events

Remote peer IP addresses

Continuous blockchain activity

These logs demonstrate that the node is actively communicating with the public Bitcoin Testnet.

Peer Count Verification

Network connectivity was quantitatively verified using:

bitcoin-cli -testnet getpeerinfo | wc -l

A peer count greater than zero confirms:

Successful peer discovery

Active participation in the testnet P2P network

Readiness for mining and block submission

On-Chain Visibility Confirmation

Because this is Bitcoin Testnet:

Blocks mined by this node will be broadcast to public peers

Transactions and blocks can be viewed on public explorers such as:

mempool.space/testnet

blockstream.info/testnet

This enables judges and reviewers to independently verify:

Block hashes

Coinbase transactions

Mining addresses

Network acceptance
